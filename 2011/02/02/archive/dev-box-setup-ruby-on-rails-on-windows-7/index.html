Ruby on Rails on Windows 7

2011-02-02

I recently set up my dev box at home with Ruby on Rails. I am running Windows 7 64-bit on my machine, and after working through a number of issues, I now have a nice dev box setup on Windows 7. Here you go, detailed instructions in case you want to replicate this!

BASIC RAILS SETUP

  • Download & install Ruby: Choose version 1.8.7-p302 (1.8.7-p330 has issues installing rails on Windows 7 64-bit; 1.9.2-p136 has issues with bundler). Here is a link to the installer: http://rubyforge.org/frs/download.php/72085/rubyinstaller-1.8.7-p302.exe. Choose to install this in the default library of c:ruby187.

  • Open a ruby command prompt & install rails: Open a ruby command prompt. Create a directory your rails projects & switch to this. Run the command to download & install rails.

    c:rails_projects> gem install rails

  • Download & install SQLite: Download SQLite - you need to download the SQLite shell (here) and the SQLite DLL’s (here). Unzip these archives, and move the files in them to c:ruby187bin.

  • Create a rails project: It’s time to create a rails test project. From the ruby command line, run these lines:

    c:rails_projects> rails new test_project

    c:rails_projects> cd test_project

    c:rails_projectstest_project> bundle install

    c:rails_projectstest_project> rails s

  • If all this goes flawlessly, then you have a basic, up and running rails installation. Verify this by going to http://localhost:3000, and you should see the Ruby on Rails welcome page.
    SETTING UP VERSION CONTROL

  • Install and set up Git: Download Git, a version control system, and install it. The right Windows binary for this is here. During the installation, few things that are important to do are -

    • Choose the check-boxes for Windows Explorer integration.
    • Choose the “Run Git from Windows Command Prompt” radio-box.

  • Create a local Git repository: From your command prompt, run the following commands to initialize and create a repository:

    c:rails_projectstest_project> git config —global user.name “Example User”

    c:rails_projectstest_project> git config —global user.email “user@example.com”

    c:rails_projectstest_project> git init

    c:rails_projectstest_project> git add .

    c:rails_projectstest_project> git commit -am “First checkin!”

  • Create a GitHub account & push your repository:

    • Go to www.github.com and sign up for a free account.
    • Create a public key - a great guide to doing this is here. You may need to drop in to Git Bash shell under Git in your start menu to run the ssh-keygen command.
    • Sign in to your GitHub account, go to “Account Settings” and “SSH Public Keys” and set your public key there.
    • Create a new GitHub repository from the home page of your Git account - you may want to name this test_project (just as the rails project you have above).
    • Run the following commands from your command line to push your local test project repository to GitHub, replacing the username & project name below.

c:rails_projectstest_project> git remote add origin git@github.com:<username/project_name.git>

c:rails_projectstest_project> git push origin master Now you have your project running, under source control and backed up to GitHub! Make it a habit to run git commit often, and then git push master to push the changes to GitHub!
*HOSTING AN APP ON HEROKU

  • Go to www.heroku.com and sign up for an account.

  • Run the following line from the command line to install the gem for Heroku & push your files to the Heroku service:

    c:rails_projectstest_project> gem install heroku

    c:rails_projectstest_project> heroku create

    c:rails_projectstest_project> git push heroku master

    c:rails_projectstest_project> heroku open

  • If all goes well, you should now see the default Ruby on Rails welcome page hosted on Heroku - your test app is now on Heroku!
    OTHER THINGS I LIKE ON MY DEV BOX!

  • Install SQLite Browser: Get this for Windows from http://sourceforge.net/projects/sqlitebrowser/. This allows you to open up and view your database files in a nice GUI.

  • Install Sublime Text editor: I love this text editor, and absolutely recommend it for Rails development - get it from http://www.sublimetext.com/.
  • Install curl: This will allow you to check the REST-fulness of your URL’s. Get it from: http://curl.haxx.se/download.html.
  • Install putty: This will allow you to FTP your files to your ISP, in case you have hosting elsewhere. Get it from: http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html.
  • IDE: NetBeans or Aptana? You could either install the NetBeans IDE for Ruby or the Aptana Studio IDE. I am not an IDE kind of guy myself, but I have heard that NetBeans works better on Windows. You can get it from here: http://netbeans.org/downloads/index.html.
    You should be all set up now with everything you need for regular, on-going Ruby on Rails development from Windows!
